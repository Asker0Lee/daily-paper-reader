## 摘要
为了缓解大语言模型（LLM）在 NPU 硬件上推理时遇到的内存墙瓶颈，并针对国产基础设施对主流投机解码算法原生支持匮乏的问题，本研究提出了一种针对 OpenPangu-7B 的端到端投机推理加速方案。

## Abstract
To mitigate the Memory Wall bottleneck encountered by Large Language Models (LLMs) during inference on \textbf{NPU} hardware, and addressing the scarcity of native support for mainstream speculative decoding algorithms on domestic infrastructure, this study presents an end-to-end speculative inference acceleration scheme for OpenPangu-7B.

### 1. 核心问题与研究背景
*   **核心问题**：大语言模型（LLM）在推理时面临“内存墙”瓶颈，即内存带宽增长远落后于计算能力，导致 NPU 的强大算力在自回归解码过程中因频繁读取权重而处于闲置状态。
*   **研究背景**：
    *   **投机解码（Speculative Decoding）**：通过轻量级模型或模块预先生成多个候选 Token，再由主模型并行验证，将内存受限的任务转变为计算受限的任务。
    *   **NPU 适配挑战**：主流投机算法（如 Medusa）依赖动态控制流和 CUDA 优化，而国产 NPU 架构（如昇腾）采用**静态图（Static Graph）**执行模式，动态的树状注意力机制会触发频繁的图重编译和同步开销，导致加速效果抵消。

### 2. 方法论
论文提出了一套针对 NPU 优化的端到端加速方案，核心技术包括：
*   **多头预测架构（Medusa Heads）**：在 OpenPangu-7B 冻结的骨干网络后添加 $K$ 个轻量级 MLP 头，直接根据当前隐藏状态预测未来多个位置的 Token 概率分布。
*   **硬件友好的静态树构建**：
    *   **张量化树拓扑**：预先计算树状结构的父子关系，并映射为静态张量（`medusa_attn_mask`），避免运行时动态构建树。
    *   **零拷贝检索策略**：构建静态查找表（`retrieve_indices`），利用 NPU 片上 `Gather` 算子直接从候选缓冲区提取接受的序列，消除 CPU 与 NPU 之间的同步开销。
*   **训练策略**：采用加权交叉熵损失函数，对越远的预测位置赋予越小的权重，以应对预测不确定性的增加。

### 3. 实验设计
*   **模型与基准**：以 **OpenPangu-Embedded-7B-V1.1** 为目标模型（34层，70亿参数）。
*   **对比硬件**：华为昇腾 NPU（搭载鲲鹏 920 CPU） vs. NVIDIA RTX A6000 (Ampere 架构)。
*   **数据集**：使用 ShareGPT 公开数据集以及通过 OpenPangu-7B 生成的**自蒸馏数据集**（Soft Labels）。
*   **评估指标**：端到端加速比（Speedup）、接受率（Accept Rate, AC）、开销比例（Overhead Ratio）。

### 4. 资源与算力
*   **硬件环境**：使用了昇腾 NPU 集群（具体型号为 910B）。

    *   使用 AdamW 优化器，学习率 1e-3，全局 Batch Size 为 64。

    *   推理测试均采用 Float16 精度。

### 5. 实验数量与充分性
*   **实验维度**：
    1.  **数据规模消融实验**：对比了 2k、10k、50k 不同样本量对预测头准确率的影响。

*   **充分性评价**：实验设计较为客观，特别是针对“自蒸馏数据中保留特殊控制 Token”的消融分析，揭示了模型对格式规范学习的重要性。但实验主要集中在 7B 规模模型，未验证更大参数规模下的表现。

### 6. 主要结论与发现
*   **加速效果**：在短序列生成任务（128步）中，该方案在 NPU 上实现了 **1.35x** 的端到端加速。
*   **自蒸馏的重要性**：使用自蒸馏数据并将 Top-1 准确率提升至 74.6%（Head 1），证明了预测头必须模仿主模型的“思维习惯”和特殊 Token 格式。
*   **长序列瓶颈**：随着序列长度增加（如达到 1024），加速比会下降。原因是 KV-Cache 增长导致内存访问模式变得稀疏且带宽密集，NPU 对此类内存受限操作比 GPU 更敏感。

### 7. 优点
*   **针对性强**：解决了投机解码在国产 NPU 静态图架构下“水土不服”的核心痛点。
*   **工程落地性高**：提出的静态树和零拷贝策略具有很强的实用价值，避开了昂贵的 JIT 重编译。
*   **深度分析**：不仅展示了加速结果，还深入探讨了内存带宽对长序列推理的限制，为后续优化指明了方向。

### 8. 不足与局限
*   **长文本挑战**：在长上下文场景下，计算开销增长较快，加速优势会逐渐消失（甚至低于 1.0x）。
*   **模型单一性**：实验仅基于 OpenPangu-7B，虽然该模型具有代表性，但未展示在其他主流开源模型（如 Llama-3）上的泛化能力。
*   **算子融合尚有空间**：论文提到未来仍需在算子融合和 KV-Cache 压缩方面做进一步优化以应对长文本。
