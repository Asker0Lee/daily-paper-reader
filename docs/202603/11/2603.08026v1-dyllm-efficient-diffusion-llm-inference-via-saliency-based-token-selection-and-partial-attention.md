## 摘要
掩码扩散语言模型 (MDLM) 支持并行 Token 解码，为自回归生成的序列化特性提供了一种极具前景的替代方案。然而，其迭代去噪过程的计算开销仍然很大，因为它在每一步都会重复处理整个序列。我们观察到，在这些扩散步骤中，大多数 Token 的表示保持稳定；只有一小部分（我们称之为显著 Token）对下一次更新有实质性贡献。利用这种时间稀疏性，我们提出了 DyLLM，这是一个无需训练的推理框架，通过选择性地仅计算这些显著 Token 来加速解码。DyLLM 通过测量相邻去噪步骤之间注意力上下文的余弦相似度来识别显著性。它仅针对显著 Token 重新计算前馈和注意力操作，而对剩余部分则重用缓存的激活值。在多种推理和代码生成基准测试中，DyLLM 实现了高达 9.6 倍的吞吐量提升，同时在很大程度上保留了 LLaDA 和 Dream 等最先进模型的基准准确度。

## Abstract
Masked Diffusion Language Models (MDLMs) enable parallel token decoding, providing a promising alternative to the sequential nature of autoregressive generation. However, their iterative denoising process remains computationally expensive because it repeatedly processes the entire sequence at every step. We observe that across these diffusion steps, most token representations remain stable; only a small subset, which we term salient tokens, contributes meaningfully to the next update. Leveraging this temporal sparsity, we present DyLLM, a training-free inference framework that accelerates decoding by selectively computing only these salient tokens. DyLLM identifies saliency by measuring the cosine similarity of attention contexts between adjacent denoising steps. It recomputes feed-forward and attention operations only for salient tokens while reusing cached activations for the remainder. Across diverse reasoning and code-generation benchmarks, DyLLM achieves up to 9.6x higher throughput while largely preserving the baseline accuracy of state-of-the-art models like LLaDA and Dream.

*   **研究背景**：掩码扩散语言模型（MDLM，如 LLaDA、Dream）通过并行 Token 解码打破了自回归（AR）模型逐个生成 Token 的限制，具有极高的生成潜力。
*   **核心痛点**：MDLM 在推理时面临“效率困境”。由于其双向注意力机制，每一步迭代去噪都需要重新处理整个序列（包括 Prompt 和已生成的 Response），这类似于在每一步都进行昂贵的“全量预填充（Prefill）”，导致计算开销（尤其是 FFN 层）随迭代次数剧增。
*   **核心动机**：作者观察到扩散步骤间存在**时间稀疏性（Temporal Sparsity）**——在连续的去噪步骤中，大部分 Token 的表示是极其稳定的，只有少数关键 Token（显著 Token）发生了实质性的语义变化。

### 2. 方法论
DyLLM 是一个**无需训练（Training-free）**的推理加速框架，核心思想是“动态识别显著 Token 并仅对其进行重计算”。
*   **层自适应显著性选择（Layer-Adaptive Saliency）**：
    *   通过计算相邻步骤 $t$ 与 $t-1$ 之间注意力上下文向量的**余弦相似度**来衡量 Token 的稳定性。
    *   若相似度低于设定阈值 $\tau$，则该 Token 被标记为“显著（Salient）”，执行完整的 FFN 计算；否则，直接复用上一轮的缓存结果。
*   **显著性感知近似注意力（Saliency-Aware Approximate Attention）**：
    *   将注意力更新分解：对于显著 Token，执行完整的行更新（Exact Row Update）；对于非显著 Token，利用显著 Token 的值变化（$\Delta V$）进行列稀疏更新（Approximate Update），从而将注意力复杂度从 $O(N^2)$ 降低到 $O(N \cdot |A|)$，其中 $|A|$ 是显著 Token 的数量。
*   **响应优先策略（Response-only Step）**：
    *   利用 RoPE 位置编码的局部性原理，观察到显著更新多集中在响应部分。DyLLM 周期性地处理全量序列，而在中间步骤仅处理响应 Token，进一步减少冗余。
