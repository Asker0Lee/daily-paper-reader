## 摘要
思维链（CoT）提高了推理的可靠性，但也增加了 token 成本，这促使人们对显式推理轨迹进行训练后压缩。然而，最短的充分推理并非通用的：它取决于任务难度、模型能力和训练状态，这使得固定长度的目标变得脆弱。在实践中，朴素的基于强化学习（RL）的压缩也可能不合时宜地缩短面向用户的答案，因为单一的生成级（completion-level）学习信号会在“思考/答案”边界发生泄漏。我们提出了难度缩放的分段式 GRPO（DSS-GRPO），它将回报分解为思考和答案两个部分，计算每个分段的组内相对优势，并通过硬 token 掩码进行路由，使得压缩更新仅作用于思考部分，而答案对齐仅作用于答案部分。DSS-GRPO 使用提示词级别的组内整形（shaping）和难度感知缩放，在不破坏答案行为的情况下鼓励简洁的推理。

## Abstract
Chain-of-thought (CoT) improves reasoning reliability but increases token cost, motivating post-training compression of explicit reasoning traces. However, the shortest sufficient reasoning is not universal: it depends on difficulty, model capacity, and training state, making fixed length targets brittle. In practice, naive RL-based compression can also undesirably shorten the user-facing answer, because a single completion-level learning signal leaks across the think/answer boundary. We propose Difficulty-Scaled Segment-Wise GRPO (DSS-GRPO), which decomposes returns into think and answer components, computes group-relative advantages per segment, and routes them with hard token masks so compression updates act only on think while answer alignment acts only on answer. DSS-GRPO uses prompt-wise within-group shaping and difficulty-aware scaling to encourage concise reasoning without collapsing answer behavior.

*   **研究背景**：思维链（CoT）显著提升了大型语言模型（LLM）的推理能力，但冗长的推理过程带来了高昂的推理延迟和 Token 成本。

### 2. 方法论：DSS-GRPO
论文提出了**难度缩放的分段式 GRPO（DSS-GRPO）**，核心思想是实现推理与答案的解耦优化：
*   **分段掩码与路由（Segment-Wise Routing）**：
    *   利用 `<think>` 和 `<answer>` 边界标记，将生成内容划分为两个分段。
    *   引入硬 Token 掩码（$M_{thk}$ 和 $M_{ans}$），确保压缩奖励仅作用于思考 Token，而答案对齐奖励仅作用于答案 Token。
*   **难度感知缩放（Difficulty-Aware Scaling）**：
    *   定义难度权重 $W_{diff} = 2 - \hat{p}_{succ}$（$\hat{p}_{succ}$ 为组内成功率）。
    *   **非对称缩放**：在困难题目上（成功率低），放大正向优势信号（Positive Advantages），引导模型向少数正确的推理路径靠拢，同时不放大负向信号以减少噪声。
*   **奖励设计**：
    *   **思考压缩奖励 ($R_{eff}$)**：采用组内最小-最大缩放（Min-Max Shaping），根据同组样本的相对长度给予奖励，避免设定全局绝对长度目标。
    *   **答案长度对齐奖励 ($R_{len}$)**：将当前答案长度与原始基座模型的参考长度进行对比，设置容差带（Tolerance Band），惩罚过短的答案，允许适度冗余。
    *   **质量门控（Quality Gate）**：仅对格式正确且答案正确的样本激活长度奖励，防止模型通过“走捷径”（如直接跳过推理）来刷分。
