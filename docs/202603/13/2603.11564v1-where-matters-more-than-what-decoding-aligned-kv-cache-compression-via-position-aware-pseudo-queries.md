## 摘要
键值（KV）缓存对于大语言模型（LLMs）的高效推理至关重要，但过长的上下文会显著增加 KV 缓存的内存占用。现有的 KV 缓存压缩方法通常依赖于预填充阶段 Prompt 观察窗口内的输入侧注意力模式来估计 Token 的重要性。由于这些评估并非源自解码过程，它们往往无法为未来的生成保留关键 Token。直观上，一个有效的观察窗口应该镜像解码阶段的查询，以准确反映生成过程将关注哪些 Token。然而，在推理过程中，真实的解码查询本质上是不可用的。为了构建伪查询来近似它们，我们发现位置信息比语义内容起着更关键的作用。基于这一见解，我们提出了通过位置感知伪查询实现的解码对齐 KV 缓存压缩（DapQ），这是一种新颖且轻量级的逐出框架，它利用位置感知伪查询来模拟输出 Token，从而建立一个有效的观察窗口进行重要性评估。它与实际的生成上下文紧密对齐，并实现了精确的 Token 逐出。在多个基准测试和大语言模型上的广泛评估表明，DapQ 取得了卓越的性能，特别是在严格的内存限制下（例如，在仅 3% 的 KV 缓存预算下，在 NIAH 上实现了高达 99.5% 的近乎无损性能）。

## Abstract
The Key-Value (KV) cache is crucial for efficient Large Language Models (LLMs) inference, but excessively long contexts drastically increase KV cache memory footprint. Existing KV cache compression methods typically rely on input-side attention patterns within a prompt observation window to estimate token importance during the prefill stage. They fail to preserve critical tokens for future generation since these assessments are not derived from the decoding process. Intuitively, an effective observation window should mirror the decoding-stage queries to accurately reflect which tokens the generation process will attend to. However, ground-truth decoding queries are inherently unavailable during inference. For constructing pseudo queries to approximate them, we find that positional information plays a more critical role than semantic content. Motivated by this insight, we propose decoding-aligned KV cache compression via position-aware pseudo queries (DapQ), a novel and lightweight eviction framework that leverages position-aware pseudo queries to simulate the output tokens, thereby establishing an effective observation window for importance assessment. It aligns closely with the actual generation context and enables precise token eviction. Extensive evaluations across multiple benchmarks and LLMs demonstrate that DapQ achieves superior performance, particularly under strict memory constraints (e.g., up to nearly lossless performance 99.5% on NIAH with 3% KV cache budgets).

### 论文总结：Where Matters More Than What (DapQ)

#### 1. 核心问题与整体含义（研究动机和背景）
随着大语言模型（LLM）处理的上下文长度不断增加，**KV Cache（键值缓存）**占用的内存呈线性增长，成为长文本推理的主要瓶颈。现有的 KV Cache 压缩方法（如 SnapKV、PyramidKV）通常在**预填充（Prefill）阶段**利用 Prompt 末尾的观察窗口来评估 Token 的重要性。
*   **研究动机**：作者发现这种“以输入为中心”的评估方式与实际“解码阶段”的查询（Query）存在**失配（Misalignment）**。理想情况下，应该用解码时的 Query 来筛选 KV，但推理时未来的 Query 是不可知的。
*   **核心发现**：论文通过实验揭示了一个关键见解——**位置信息（Where）对 Query 向量表示的影响远大于语义内容（What）**。这意味着即使不知道未来要生成什么字，只要知道它们的位置，就能构造出高质量的“伪查询”来模拟解码过程。

#### 2. 方法论：DapQ 框架
作者提出了 **DapQ**（Decoding-aligned KV Cache Compression via Position-aware Pseudo Queries），其核心流程如下：
*   **构造伪 Token**：在原始 Prompt（长度为 $L_p$）后拼接 $N$ 个伪 Token（内容可以是随机采样或首尾拼接）。
*   **位置感知分配**：关键步骤是为这些伪 Token 分配**未来解码阶段的位置 ID**（即 $L_p$ 到 $L_p + N - 1$），而不是紧随其后的物理位置。
*   **生成伪查询（Pseudo Queries）**：在预填充阶段，模型处理扩展序列，获取这些伪 Token 对应的 Query 向量 $Q_{pseudo}$。
*   **重要性评估与剔除**：
    1.  计算 $Q_{pseudo}$ 与 Prompt 中所有 Key 的注意力分数。
    2.  对分数进行累加，得到每个 Prompt Token 的重要性得分 $S(j)$。
    3.  保留得分最高的 Top-K 个 KV 对，剔除其余部分以及所有的伪 Token。
*   **解码对齐**：完成剔除后，模型从位置 $L_p$ 开始正式解码，此时的 KV 缓存已根据未来可能的需求进行了精准优化。

#### 6. 主要结论与发现
*   **位置主导性**：Query 向量的表示受位置编码（RoPE）支配，语义内容的改变对注意力模式的影响相对较小。
*   **性能卓越**：DapQ 在极低缓存预算下表现惊人。例如在 LLaMA-3-8B 上，仅保留 256 个 KV（约 3% 预算），在 NIAH 任务上达到 99.5% 的准确率，远超 SnapKV 等方法。
*   **位置偏移敏感性**：伪查询的位置必须精确对齐未来解码位置，任何偏移都会导致性能显著下降。

#### 7. 优点与亮点
*   **轻量且高效**：无需像 LAQ++ 那样预生成真实的响应文本，仅通过调整位置 ID 即可实现对齐，不增加内存峰值。
*   **即插即用**：作为一种剔除策略，它可以直接应用于现有的 Transformer 架构模型，无需微调。
*   **理论支撑**：论文不仅有实验观察，还提供了数学证明，解释了为什么高相似度的 Query 能产生相似的注意力分布。

#### 8. 不足与局限
*   **语义仍有微弱影响**：虽然位置是主导，但实验显示语义内容完全无关时（如乱码），性能虽稳定但仍有提升空间。
*   **层间差异未优化**：目前 DapQ 在所有层使用统一的伪查询策略，未考虑不同 Transformer 层对位置和语义敏感度的差异。
*   **超参数依赖**：伪查询的长度 $N$ 需要根据任务微调，过长会导致信号稀释（Dilution Effect）。
