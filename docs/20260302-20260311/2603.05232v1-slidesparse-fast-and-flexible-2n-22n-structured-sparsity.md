## 摘要
NVIDIA 的 2:4 稀疏张量核心（Sparse Tensor Cores）可提供 2 倍吞吐量，但要求严格的 50% 剪枝率，这一比例会导致大语言模型（LLM）的推理准确率大幅下降（如 Qwen3 从 54% 降至 15%）。更温和的 (2N-2):2N 模式（如 6:8，即 25% 剪枝）能保持准确率，但缺乏硬件支持，只能回退到稠密执行，无法从稀疏性中获益。我们提出了 SlideSparse，这是首个在通用 GPU 上为 (2N-2):2N 模型系列解锁稀疏张量核心加速的系统。我们的滑动窗口分解（Sliding Window Decomposition）在不损失准确率的情况下，将任何 (2N-2):2N 权重块重构为 N-1 个重叠的符合 2:4 规范的窗口；激活提升（Activation Lifting）则以近乎零的成本将相应的激活重排融合到逐 token 量化中。SlideSparse 已集成到 vLLM，并在多种 GPU（A100、H100、B200、RTX 4090、RTX 5080、DGX-spark）、精度（FP4、INT8、FP8、BF16、FP16）和模型系列（Llama、Qwen、BitNet）上进行了评估。在计算受限的工作负载下，Qwen2.5-7B 在 6:8 权重稀疏度下的实测加速比（1.33x）接近理论上限 N/(N-1)=4/3，证明了 (2N-2):2N 是实现保持准确率的 LLM 加速的一条切实可行的路径。代码开源于 https://github.com/bcacdwk/vllmbench。

## Abstract
NVIDIA's 2:4 Sparse Tensor Cores deliver 2x throughput but demand strict 50% pruning -- a ratio that collapses LLM reasoning accuracy (Qwen3: 54% to 15%). Milder $(2N-2):2N$ patterns (e.g., 6:8, 25% pruning) preserve accuracy yet receive no hardware support, falling back to dense execution without any benefit from sparsity. We present SlideSparse, the first system to unlock Sparse Tensor Core acceleration for the $(2N-2):2N$ model family on commodity GPUs. Our Sliding Window Decomposition reconstructs any $(2N-2):2N$ weight block into $N-1$ overlapping 2:4-compliant windows without any accuracy loss; Activation Lifting fuses the corresponding activation rearrangement into per-token quantization at near-zero cost. Integrated into vLLM, SlideSparse is evaluated across various GPUs (A100, H100, B200, RTX 4090, RTX 5080, DGX-spark), precisions (FP4, INT8, FP8, BF16, FP16), and model families (Llama, Qwen, BitNet). On compute-bound workloads, the measured speedup ratio (1.33x) approaches the theoretical upper-bound $N/(N-1)=4/3$ at 6:8 weight sparsity in Qwen2.5-7B, establishing $(2N-2):2N$ as a practical path to accuracy-preserving LLM acceleration. Code available at https://github.com/bcacdwk/vllmbench.

# SlideSparse: 快速且灵活的 (2N-2):2N 结构化稀疏论文总结

## 1. 核心问题与整体含义（研究动机和背景）
论文的核心问题在于解决 **NVIDIA 2:4 结构化稀疏硬件支持与大语言模型（LLM）精度需求之间的矛盾**。
*   **背景**：NVIDIA 的 Ampere 及后续架构支持 2:4 稀疏加速，可提供 2 倍吞吐量，但强制要求 50% 的剪枝率。
*   **痛点**：对于 LLM（尤其是推理任务），50% 的剪枝率往往会导致精度灾难性下降（如 Qwen3 准确率从 54% 崩塌至 15%）。
*   **动机**：更温和的稀疏模式（如 6:8，即 25% 剪枝）能很好地保持精度，但现有硬件（Tensor Core）不支持这种格式，导致这些模型在部署时只能回退到稠密计算，无法获得任何加速。SlideSparse 旨在通过软件算法创新，在现有硬件上解锁 (2N-2):2N 模式的加速能力。

## 2. 方法论
SlideSparse 的核心思想是**计算套利（Computational Arbitrage）**：通过适度的数据扩张来换取硬件兼容性。

### 核心技术细节：
1.  **滑动窗口分解（Sliding Window Decomposition）**：
    *   **原理**：证明了任何一个 (2N-2):2N 的权重块都可以无损地分解为 $N-1$ 个重叠的、符合 2:4 规范的窗口。
    *   **实现**：例如 6:8 模式（N=4），使用 3 个步长为 2、大小为 4 的重叠窗口。重叠区域充当“溢出缓冲区”，当一个窗口达到 2 个非零值的容量上限时，多余的非零值会落入下一个窗口。
    *   **扩张因子**：扩张因子 $\gamma = 2 - 2/N$。只要硬件提供的 2 倍加速大于 $\gamma$，就能获得净加速。

2.  **激活提升（Activation Lifting）**：
    *   为了匹配权重的窗口分解，输入激活向量需要进行相应的索引重排和复制。
    *   **融合内核**：SlideSparse 将激活提升操作融合进“逐 token 量化（per-token quantization）”内核中。在将激活从高精度转为 INT8/FP8 并存入显存的过程中，直接按滑动窗口的索引进行写入，从而以近乎零的额外访存成本完成数据重排。

3.  **系统集成**：
    *   设计了三阶段流水线：离线权重打包器、模型加载时的压缩初始化、以及在线推理时的融合算子。

## 3. 实验设计
*   **集成框架**：集成到主流推理引擎 **vLLM** 中。
*   **测试模型**：Llama 3.2 (1B/3B), Qwen 2.5 (7B/14B), 以及 BitNet 1.58 (2B)。
*   **对比基准（Benchmark）**：

*   **测试场景**：涵盖了 **Prefill（预填充，计算受限）** 和 **Decode（解码，访存受限）** 两个阶段。
*   **稀疏模式**：测试了 4:6, 6:8, 8:10 等多种 (2N-2):2N 模式。

## 6. 主要结论与发现
1.  **加速效果显著**：在计算受限的 Prefill 阶段，6:8 模式在 Qwen2.5-7B 上实现了 **1.33x** 的实测加速，完全达到了理论上限 $N/(N-1)$。
2.  **精度-速度平衡**：(2N-2):2N 模式成功填补了“高精度稠密”与“低精度 2:4 稀疏”之间的空白。6:8 模式在保持接近稠密模型精度的同时，获得了实质性的硬件加速。
3.  **跨架构有效性**：SlideSparse 在从 Ampere 到 Blackwell 的各代 NVIDIA GPU 上均表现出一致的加速趋势。
4.  **高效融合**：通过将激活提升与量化融合，SlideSparse 的实际效率经常超过 100%（相对于原生 2:4 路径），证明了其系统设计的优越性。

## 7. 优点
*   **硬件零修改**：完全基于现有商用 GPU 的 Tensor Core，无需等待新硬件。
*   **数学无损**：滑动窗口分解是数学等价的，不会引入额外的近似误差。
*   **系统集成度高**：直接集成到 vLLM，易于在生产环境中部署。
*   **理论严密**：提供了通用的滑动窗口分解证明，适用于未来可能的 M:N 硬件。

*   **小 Batch 限制**：在 Decode 阶段（Batch size 较小时），由于稀疏元数据处理的固定开销，加速效果不如 Prefill 阶段明显。
