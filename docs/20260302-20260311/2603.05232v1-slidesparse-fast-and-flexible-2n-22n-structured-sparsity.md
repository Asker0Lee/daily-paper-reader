## 摘要
NVIDIA 的 2:4 稀疏 Tensor Core 提供了 2 倍的吞吐量，但要求严格的 50% 剪枝——这一比例会导致大语言模型（LLM）的推理准确率大幅下降（如 Qwen3 从 54% 降至 15%）。更温和的 (2N-2):2N 模式（例如 6:8，25% 剪枝）能保持准确率，但由于缺乏硬件支持，只能回退到稠密执行，无法从稀疏性中获益。我们提出了 SlideSparse，这是首个在商用 GPU 上为 (2N-2):2N 模型系列解锁稀疏 Tensor Core 加速的系统。我们的滑动窗口分解（Sliding Window Decomposition）在不损失准确率的情况下，将任何 (2N-2):2N 权重块重构为 N-1 个重叠的、符合 2:4 规范的窗口；激活提升（Activation Lifting）则以近乎零的成本将相应的激活重排融合到逐 token 量化中。SlideSparse 已集成到 vLLM 中，并在多种 GPU（A100、H100、B200、RTX 4090、RTX 5080、DGX-spark）、精度（FP4、INT8、FP8、BF16、FP16）和模型系列（Llama、Qwen、BitNet）上进行了评估。在计算密集型工作负载中，Qwen2.5-7B 在 6:8 权重稀疏度下的实测加速比（1.33x）接近理论上限 N/(N-1)=4/3，证明了 (2N-2):2N 是实现保持准确率的 LLM 加速的一条切实可行的路径。代码可在 https://github.com/bcacdwk/vllmbench 获取。

## Abstract
NVIDIA's 2:4 Sparse Tensor Cores deliver 2x throughput but demand strict 50% pruning -- a ratio that collapses LLM reasoning accuracy (Qwen3: 54% to 15%). Milder $(2N-2):2N$ patterns (e.g., 6:8, 25% pruning) preserve accuracy yet receive no hardware support, falling back to dense execution without any benefit from sparsity. We present SlideSparse, the first system to unlock Sparse Tensor Core acceleration for the $(2N-2):2N$ model family on commodity GPUs. Our Sliding Window Decomposition reconstructs any $(2N-2):2N$ weight block into $N-1$ overlapping 2:4-compliant windows without any accuracy loss; Activation Lifting fuses the corresponding activation rearrangement into per-token quantization at near-zero cost. Integrated into vLLM, SlideSparse is evaluated across various GPUs (A100, H100, B200, RTX 4090, RTX 5080, DGX-spark), precisions (FP4, INT8, FP8, BF16, FP16), and model families (Llama, Qwen, BitNet). On compute-bound workloads, the measured speedup ratio (1.33x) approaches the theoretical upper-bound $N/(N-1)=4/3$ at 6:8 weight sparsity in Qwen2.5-7B, establishing $(2N-2):2N$ as a practical path to accuracy-preserving LLM acceleration. Code available at https://github.com/bcacdwk/vllmbench.

*   **核心问题**：更温和的稀疏模式（如 6:8，即 25% 剪枝）能够很好地保持精度，但目前主流 GPU 硬件并不直接支持这些模式，导致这些模型在推理时只能回退到稠密计算，无法获得任何性能收益。
*   **整体含义**：SlideSparse 旨在通过算法创新，在不修改硬件的前提下，利用现有的 2:4 硬件加速能力来运行 (2N-2):2N 稀疏模式，填补“高精度-无加速”与“低精度-高加速”之间的空白。

## 2. 方法论：核心思想与关键技术
SlideSparse 的核心思想是**“计算套利”**：通过适度的数据膨胀来换取硬件兼容性。
*   **滑动窗口分解（Sliding Window Decomposition, SWD）**：
    *   将一个 (2N-2):2N 的权重块分解为 $N-1$ 个重叠的、符合 2:4 规范的窗口。
    *   例如，对于 6:8 模式（N=4），使用 3 个步长为 2、大小为 4 的滑动窗口。重叠区域充当“溢出缓冲区”，当一个窗口达到 2 个非零值的容量上限时，多余的非零值会落入下一个窗口。
    *   该方法在数学上是等价且无损的，权重膨胀因子为 $\gamma = 2 - 2/N$。
*   **激活提升（Activation Lifting）**：
    *   为了匹配权重的窗口分解，输入激活也需要进行相应的索引重排（复制）。
    *   **关键优化**：将激活重排操作融合进 LLM 推理必经的“逐 token 量化（per-token quantization）”内核中。利用 Triton 编写的融合算子，在量化写入内存时直接完成重排，额外开销几乎为零。
*   **系统集成**：
    *   设计了三阶段流水线：离线权重打包、模型加载时的压缩、在线融合执行。
    *   无缝集成到 vLLM 推理框架中，作为自定义量化后端运行。
